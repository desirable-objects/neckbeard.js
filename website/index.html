What is it?

Oh! you can create iterative jasmine specifications by setting up a few 'where' clauses for your test data, you need only write your test code once.

Supposing that you had a simple function:

						
function(x, y) {
	return x+y;
};

You probably want to test that it adds the numbers 1 + 1 and returns 2. You might also want to test what happens if you add 1 + 0. Then you have to consider -1 + 1, 0 + -1, 0 + 0, and so on...

Why is it?

Your question doesn't really make sense, but the reason it exists is to help your test code be as maintainable, readable, and conform as well to the DRY principle as your production code.

What is it like?

I knew you'd ask that. It's a bit like Spock and Geb.

How do I use it?

You just don't give up, do you? Here is how it is done:

						
describe('spec which loops', function() {

  describe('Here is some functionality', function() {

    neckbeard
    .iterate('first', 'second', 'sum')
    .where(   3,       3,        6,
              10,      4,        14,
              7,       1,        8
    );

    neckbeard.loop('#firstNumber plus #secondNumber is #sum', function(first, second, sum) {
      expect(first+second).toEqual(sum);
    });
  });
});
